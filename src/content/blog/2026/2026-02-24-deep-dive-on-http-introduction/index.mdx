HTTP is the foundation of every single web application that has ever existed. It is the protocol that allows clients (like web browsers) to communicate with servers and retrieve resources such as HTML documents, images, and APIs. 

Every developer will inevitably have to work with HTTP at some point in their career, whether it's making API calls, handling form submissions. That said, HTTP is more than just methods and status codes.

The purpose of this series is to provide a comprehensive understanding of the HTTP protocol, in this article we will begin by giving an overview of the components of an HTTP request and response.
